**Git y GitHub explicados en español, sin miedo y sin rodeos.**

Commitea nace de una idea simple: cuando empecé a programar, entender Git y GitHub me costó más de lo que debería. No porque sea complicado, sino porque la mayoría de recursos buenos están en inglés, o dan por hecho cosas que a un principiante no le quedan claras.

Este repositorio es una explicación clara, en español, pensada tanto para quien empieza de cero como para quien ya usa Git pero se atasca en momentos concretos (un conflicto de merge, un commit en la rama equivocada, un push del que se arrepiente...).

No es un proyecto cerrado. Va a ir creciendo con el tiempo, sobre todo la sección de dudas frecuentes, que se irá llenando con preguntas reales de gente real.

## Cómo está organizado

```
commitea/
├── 01-fundamentos/          Qué es Git, qué es GitHub, tu primer repo
├── 02-ciclo-basico/         Staging, commits, historial
├── 03-ramas/                Crear, movernos, fusionar ramas
├── 04-colaboracion-github/  Forks, pull requests, code review
├── dudas-frecuentes/        Casos concretos que te atascan, sin orden fijo
└── buenas-practicas/        Para cuando ya controlas lo básico
```

Si nunca has usado Git, te recomiendo seguir el orden de las carpetas numeradas, de la 01 a la 04, como si fuera un curso.

Si ya sabes lo básico y solo buscas resolver una duda concreta, ve directo a `dudas-frecuentes/`.

## Cómo está escrito cada tema

Cada archivo sigue siempre la misma estructura, para que sepas qué esperar:

1. **Qué te vas a llevar** — el concepto en una frase.
2. **Explicación** — corta, clara, sin rodeos.
3. **Ejemplo real** — comandos reales con el resultado esperado.
4. **Cómo se rompe esto** — el error típico que se comete.
5. **Vídeo relacionado** — si existe contenido en vídeo sobre el tema.

## Contribuir
